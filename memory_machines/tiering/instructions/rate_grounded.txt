## Rating Scale (T0-T3)

**T3 - Ready to Review**: The memory prompt is well-targeted to the user's likely interest and well-constructed for spaced repetition. It can be used as-is in a review system. The question is specific, unambiguous, and will consistently trigger the same retrieval pathway. The answer captures a meaningful detail worth remembering long-term.

**T2 - Needs Polish**: The memory prompt is semantically aligned with the user's interest and could be reviewed in its current form, but would benefit from minor tweaks to wording, phrasing, or context. The core targeting and construction are sound, but small improvements would make it more effective.

**T1 - Needs Refactor**: The memory prompt is roughly in the right region of meaning but is ineffective for review in its current form. It requires restructuring. Common issues include: the question is too vague or ambiguous (soliciting multiple valid responses), lacks sufficient contextual cues for reliable recall, or lacks the depth/specificity needed for worthwhile long-term review.

**T0 - Off-Target**: The memory prompt focuses on details that are not aligned with the user's likely interest. It may be well-constructed as a flashcard, but it targets the wrong information—either focusing on an entirely different aspect of the passage (macro misalignment) or being in the general vicinity but entailing details the user doesn't care about (micro misalignment).

## Key Evaluation Criteria

A good memory prompt must satisfy two criteria:
1. **Meaningful**: It captures details the user genuinely cares about long-term (based on what they highlighted)
2. **Reliably Recallable**: It can be consistently retrieved from memory even after months, with unambiguous questions and sufficient context

Poor targeting means the prompt doesn't align with the user's interest. Poor construction means the prompt won't work well in spaced repetition (too vague, lacks context, or has multiple valid answers).

**Focus on the Question, Not the Answer**

The stringency of evaluation should be applied primarily to the **question**, as this is what cues recall. Answers can contain auxiliary details, context, or elaboration—this is fine and often helpful. What matters is whether the question reliably points to a specific retrieval target.

When distinguishing T1 from T2, focus on these question-quality checks:

- **What exactly is being asked?** Is the retrieval target that the question is cueing clear and specific, or could the question legitimately elicit multiple different answers? If not, this is T1.
- **Does the question provide sufficient cues?** Does it include enough context, specificity, or framing to reliably trigger the intended answer, or is the connection too loose or ambiguous? Pay special attention to whether contextual anchors (like named systems, frameworks, or papers) are preserved or lost—replacing a specific anchor with a generic domain reference degrades the question's ability to cue reliable recall.
- **Is the language source-anchored?** "Specific" means source-anchored, not merely detailed. A prompt using unique phrases from the source text is more specific than one that genericizes the language. Replacing source language with generic paraphrases is a T1 failure, even if the meaning seems equivalent.
- **Is the question too generic or definitional?** Generic questions like "what is X?" or "what is the key principle of X?" are often too vague to reliably cue a specific answer. They may seem semantically aligned but fail the reviewability test. Compare this to more specific, contextual questions like "what advantage does X have over Y?" which provide stronger retrieval cues.

A prompt can be semantically on-target but still be T1 if the question fails these checks.

## Calibration: Past Ratings Are Precedents

Treat the provided `<past_ratings>` as **binding precedents**. Your goal is to replicate the user's specific judgment patterns, even if they differ from general "good flashcard" advice.

1. **Look for Structural Matches**: Scan the past ratings for prompts with similar structures to the current one.

2. **The Specificity Delta**: Directly compare the new prompt to the highest-rated past example. The key principle: past ratings reveal what type of question matches what type of user interest. 

Before comparing specificity, infer what the user cared about—was it a raw fact, a causal mechanism, specific actors, or an implication the passage illustrates? Then find past examples targeting similar interests. If highly-rated examples use a particular question style (factual, meta-textual, comparative) for that type of interest, the new prompt should match that style.

Once you've identified the appropriate comparison examples, use their verbosity and source-anchored language to guide your judgment on construction quality.

**Rule of Thumb:** If the new prompt looks like a T0/T1 example from the history, rate it T0/T1. T2 means it asks for the same fact as T3 and the user would happily review it as-is—fixes are word-level, not structural.

   - Near-complete rewrite (wrong focus, wrong predicate) → T1
